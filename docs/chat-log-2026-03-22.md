## Session 2026-03-22 [03:00-06:00] — Defender XDR RBAC Audit Tool

### Decisões Tomadas
1. **Caso de uso**: Cliente quer auditar criação/edição de RBAC no Defender XDR e alertar o SOC
2. **Abordagem**: Script PowerShell que mapeia RBAC + executa KQL + gera HTML report
3. **Fontes de dados**: CloudAppEvents (roles/grupos cloud), IdentityDirectoryEvents (AD on-prem), Graph API (roleManagement/defender)
4. **ActionTypes validados em lab**: "Add member to role.", "Remove member from role.", "Add member to group.", "Remove member from group.", "AddRole", "EditRole", "DeleteRole"
5. **Correção crítica**: `parse_json(tostring(RawEventData.ModifiedProperties[1])).NewValue` em vez de `tostring(RawEventData.ModifiedProperties[1].NewValue)` para extrair nomes de roles
6. **Correção PS 5.1**: UTF-8 BOM obrigatório, sem em-dash, sem `??`, sem ternários, `[IO.File]::WriteAllText` para HTML
7. **Correção AccountType**: Removido filtro `AccountType == "Regular"` que excluía eventos legítimos
8. **Achado RBAC**: API `roleManagement/defender` (Graph beta) retorna custom roles + assignments com `appScopeIds` e `principalIds`
9. **Achado escopo**: `appScopeIds: ["/"]` = acesso global a todos os workloads
10. **Achado caminhos**: Um principal pode ter múltiplos caminhos de acesso (role direta + grupo RBAC)

### Problemas Encontrados
1. SVG fonts muito grandes — corrigido (9→7px)
2. IDs mascarados inúteis para analista — corrigido com caminhos de acesso
3. Em-dash causa parse error PS 5.1 — corrigido
4. `$pid` conflita com variável automática PS — renomeado para `$pid2`
5. `Out-File -Encoding UTF8` no PS 5.1 pode double-encode — trocado por `[IO.File]::WriteAllText`
6. KQL com `let`+`union` falha no portal do Defender — trocado por `union()` inline
7. Seções fora de ordem UX — reorganizado (RBAC primeiro, depois eventos, depois Detection Rule)
